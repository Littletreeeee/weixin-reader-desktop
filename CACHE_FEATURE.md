# 📚 书籍缓存与离线阅读功能

**版本:** v0.9.0+  
**功能:** 自动缓存书籍内容，支持离线阅读

---

## ✨ 功能特点

### 自动缓存
- ✅ 自动缓存所有 weread.qq.com 的 GET 请求
- ✅ 100MB 缓存空间限制
- ✅ 使用 IndexedDB 存储，浏览器跨会话保留
- ✅ 自动清理最旧的缓存（满容时删除20%最老文件）

### 离线阅读
- ✅ 断网时自动从缓存读取内容
- ✅ 自动检测网络状态
- ✅ 网络恢复时自动同步

### 缓存管理
- ✅ 实时缓存统计（大小、数量）
- ✅ 按书籍清除缓存
- ✅ 一键清除全部缓存

## 🎯 工作原理

```
┌─────────────────────────────────────┐
│     用户访问微信读书                │
└────────────┬────────────────────────┘
             │
             ▼
┌─────────────────────────────────────┐
│  CacheInterceptor 拦截 fetch 请求  │
└────────────┬────────────────────────┘
             │
             ├─► 在线? ──► 网络请求
             │             │
             │             ├─► 响应 OK?
             │             │   │
             │             │   ├─► 是 ──► 缓存响应
             │             │   │         │
             │             │   └─► 否 ──► 使用缓存
             │             │
             │             └─► 请求失败 ──► 使用缓存
             │
             └─► 离线? ──► 使用缓存 ──► 无缓存? ──► 错误
                                         │
                                         └─► 有缓存 ──► 返回

                         │
                         ▼
            ┌──────────────────────────┐
            │     IndexedDB 存储       │
            │  (100MB, 跨会话保留)     │
            └──────────────────────────┘
```

## 🔧 使用说明

### 自动功能（无需配置）

1. **打开应用**
   - 缓存系统自动启动
   - 自动拦截网络请求并缓存

2. **正常阅读**
   - 所有访问的内容自动缓存
   - 无需手动操作

3. **离线阅读**
   - 断网后自动从缓存读取
   - 完全无缝体验

### 手动操作

#### 查看缓存统计
打开浏览器控制台 (F12)，输入：
```javascript
const stats = await CacheManager.getInstance().getCacheStats();
console.log(stats);
// 输出:
// {
//   totalSize: 51234567,  // 字节
//   entryCount: 123,      // 缓存项数
//   isOnline: true        // 网络状态
// }
```

#### 清除所有缓存
```javascript
await CacheManager.getInstance().clearAllCache();
```

#### 清除特定书籍缓存
```javascript
await CacheManager.getInstance().clearBookCache('bookId123');
```

## 🚀 场景示例

### 场景1: 日常阅读
```
用户流程:
1. 打开微信读书，登录
2. 打开一本书开始阅读（所有内容自动缓存）
3. 正常翻页阅读
4. 系统自动缓存每个访问的页面

离线使用:
1. 断开网络（飞行模式或关闭 WiFi）
2. 继续打开之前阅读的书籍
3. 所有内容从缓存读取，无任何差异
```

### 场景2: 出行阅读
```
准备阶段:
1. 在家用 WiFi 打开要读的书
2. 翻看几章内容，缓存下来
3. 出门前关闭应用

出行阶段:
1. 地铁/飞机上打开应用
2. 打开同一本书
3. 之前缓存的内容可离线阅读
4. 新访问的内容无法加载（网络不可用）
```

### 场景3: 网络不稳定
```
正常情况:
1. 网络波动导致请求失败
2. 系统自动从缓存返回之前的内容
3. 用户无感知网络问题

提示:
1. 浏览器控制台会记录日志
2. 缓存内容标记为 X-From-Cache: true
```

## ⚙️ 高级配置

### 修改缓存大小

编辑 `cache_manager.ts`：
```typescript
private cacheSize = 200 * 1024 * 1024; // 改为 200MB
```

### 修改自动清理触发阈值

编辑 `cache_manager.ts`：
```typescript
const toDelete = Math.ceil(allEntries.length * 0.3); // 改为删除 30%
```

### 禁用缓存

在 `inject.ts` 中注释掉缓存初始化：
```typescript
// const cacheInterceptor = new CacheInterceptor();
```

## 🐛 故障排查

### 缓存不生效
1. 检查浏览器是否支持 IndexedDB（大多数现代浏览器支持）
2. 查看浏览器控制台日志
3. 确保请求是 weread.qq.com 域名

### 缓存数据丢失
1. 检查浏览器隐私模式（无痕模式下无法保留）
2. 检查浏览器存储限制
3. 查看是否被清理工具删除

### 离线无法阅读
1. 确认之前在线时有访问过该内容
2. 检查缓存统计确认有数据
3. 查看浏览器控制台错误信息
